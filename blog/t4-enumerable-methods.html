Enumerable Methods

Enumerable Methods

July 12th, 2015

Enumerables inn Ruby are a way of going through a set of data one value at a time. The Enumerable class has an element called find_all that I believe exemplifies not only what an enumerable does but also a core tennant of Ruby theory.

According to the Ruby Docs: Enumerables#find_all returns an array containing all elements of of enum(that is, the thing being operated on) for which the given block returns a true value. Basically what that means is that you are able to give find_all a set of requirements inside of the block and have it return an array of everything that meets that requierment. For example, take a look at the following expression: 

["red", "white", "blue", "gold", "silver"].find_all {|i| i.size > 4}   #=> ["white", "silver"]
This piece of code will return an array that contains the strings "white" and "silver" because both of those words have a size greater than 4. This is but a simple example of how useful find_all can be.

Furthermore, the find_all method actually has a counterpart - reject. reject does the exact same thing as find_all except instead of returning an array of values returned a true value, it returns an array for all values that returned a false value. If we use the same array as before but change the method, we would see: 

["red", "white", "blue", "gold", "silver"].reject {|i| i.size > 4}   #=> ["red", "blue", "gold"]

One of my favorite things about Enumerables#find_all and Enumerables#reject is that they does exactly what they say they will do - they either find all instances or reject all instances that meet your requirements. One of Ruby's greatest strengths as a coding language is that the methods do what you think they would do. You don't have to guess what find_all and reject are going to do; they are highly intuitive. These are wonderful methods to represent Ruby.
