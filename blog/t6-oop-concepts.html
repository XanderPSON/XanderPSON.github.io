Variable Scope

July 26th, 2015

Scope is the concept of where variables exist and are able to be accessed. It is imperative that every coder fully understands the different varities of scope, as it affects every single variable that you will ever create. A strong working knowledge of variables is essential to mastering the concept of class and will help programmers DRY their code out. There are 5 different kinds of scope. The variables that has the largest scope are also the ones that are used the least, but we will start there so we can "zoom in" on the variables that are most commonly used. The 5 different kinds of variables are, in order of largest scope to smallest scope: Global: $variable Class: @@variable Instance: @variable Local: variable Block: variable

Global: $variable

Global variables are defined by starting the variable name with $. Think of global variables as being able to walk through walls; they can be accessed anywhere, from within a simple loop to deep inside of a class. Although this sounds fantastic and extremely useful, global variables are actually very rarely used in code. Because they are able to accessed and changed anywhere, they're very susceptible to being changed, which means that they aren't always reliable, especially in much longer code. In general variables are only used in very specific situations.

Class: @@variable

Class variables are defined by starting the variable name with @@. Class variables are able to accessed by every instance of that class. Therefore, class variables are variables that every instance of that class will have. For example, a class Human could have the class varibale @@heart = 1. Even if each instance of Human has a different name, different age, or different eye color, one thing is for certain; we all have a heart!

Instance: @variable

Instance variables are defined by starting the variable name with @. They are variables that are unique to that particular instance of a given class. For example, in our class Human, we have have an @name = "Xander". @name is an appropriate instance variable because the name of a person changes with each person. @age or @eye_color would also be good instance variables.

Local: variable

Local variables are defined by starting the variable name with a letter. A local variables' scope is limited to that of the current environment. Local variables, along with block variables, are the most common kind of variable. Local variables are the closest thing to "default" variables, as they're usually what we use when we explain the concept of a variable to a beginner. For example, x is a variable in x = 5. my_array is also a variable in my_array = [1,2,3]. These variables do not carry over to other instances of their enviornment like class or instance variables do.

Block: variable

Block variables are defined by starting the name with a letter, much like a local variable. The difference between a local and block variable is that a block variable is defined within a block and only exists in that block. Block variables are able to overwrite variables with the same name that exist on the outside, if only temporarily. Block variables are isolated and disappear after the block ends.
